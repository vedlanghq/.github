# Lexum Code of Conduct

Lexum is an engineering-focused open source project exploring deterministic
control-plane programming models.

We aim to maintain a respectful, focused, and intellectually honest
collaboration environment.

## Expected Behaviour

Participants are expected to:

- Communicate with clarity and professionalism
- Critique ideas, not individuals
- Stay technically grounded in discussions
- Provide constructive feedback
- Respect differing experience levels
- Assume good intent from other contributors

## Unacceptable Behaviour

The following behaviours are not tolerated:

- Personal attacks or harassment
- Dismissive or hostile communication
- Discriminatory language or exclusionary conduct
- Repeated off-topic disruption of technical discussions
- Publishing private information without consent

## Enforcement

Project maintainers may:

- request clarification or behavioural adjustments
- moderate or lock discussions
- remove content that violates this code
- restrict participation in severe or repeated cases

## Scope

This Code of Conduct applies to:

- GitHub issues and pull requests
- discussions and design threads
- documentation contributions
- any project-related communication channels

## Reporting

Concerns may be reported to the project maintainers through GitHub or other
official contact channels.

All reports will be reviewed in good faith.
